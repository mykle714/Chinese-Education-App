# Example Sentence — Form Modification

> ↑ Part of [EXAMPLE_SENTENCES.md](./EXAMPLE_SENTENCES.md).

How the example-sentence segment popup shows the **contextually inflected English
gloss** of a word (the right tense / number / part-of-speech form for *this*
sentence) instead of the word's bare dictionary definition.

When a user hovers/taps a segment inside an example sentence (the **est** tab), the
popup definition is adapted to how the word is used in that sentence: a verb in a
past-tense sentence shows "stuck" not "stick"; a plural noun shows "books" not
"book". This document covers the data, the runtime selection logic, and the
generation pipeline that feeds it.

> Scope: **Chinese (`zh`) only.** The Spanish enrichment path
> (`enrichSpanishExampleSentencesMetadataBatch`) attaches only a base `definition`
> per token and does **not** attach `wordForms`, so form modification does not run
> for `es`. Adding it there requires an `es` word-forms backfill plus including
> `wordForms` in the Spanish enrichment.

---

## The gloss system (`wordForms` + form modification)

The segment popup's inflected English gloss comes from **`wordForms` + runtime form
modification**: `wordForms` is a **dictionary-entry column**, and selection happens
**at runtime** when a segment is tapped, picking the correct inflected form on demand
from the tapped word's own dictionary entry.

> Historical note: an example sentence used to also carry a static, AI-written
> `segmentGloss` (an ordered per-segment "broken English" reading). It had **no
> runtime consumer** and was removed in `backfill-example-sentences.js`
> `SCRIPT_VERSION 4`, replaced on the sentence jsonb by a `sense` label (which
> meaning the sentence demonstrates — see [EXAMPLE_SENTENCES.md](./EXAMPLE_SENTENCES.md)).
> Do not confuse `sense` with form modification: `sense` is static text tagging the
> word's *meaning*; form modification is a runtime lookup of the word's inflected *form*.

## The three layers

### 1. Data layer — `wordForms` (the form inventory)

`wordForms` is a jsonb column on `dictionaryentries_zh` (and `_es`, unused). Per
**headword**, it holds an AI-generated map of English inflected forms:

```json
{"past": "ran", "present": "runs", "future": "will run", "gerund": "running"}
{"noun": "book", "noun_plural": "books"}
{"adjective": "fast", "adverb": "quickly"}
```

Allowed keys (`ALLOWED_KEYS` in the backfill script):
`past`, `present`, `future`, `gerund`, `adverb`, `adjective`, `noun`, `noun_plural`.

Only keys relevant to the entry's `partsOfSpeech` are present. An entry with no
applicable forms (classifier, conjunction, …) is stored as `{}` so it is not
re-processed; at runtime it falls back to the base definition.

Generated by **`server/scripts/backfill/chinese/backfill-word-forms.js`**
(Sonnet). The prompt extracts the base English word from `definitions[0]`, then
emits forms gated by POS — including the irregular-aware `noun`/`noun_plural` pair
(child→children, water→water for mass nouns). Guarded by
`WHERE "wordForms" IS NULL`, so re-running to add a new key (e.g. `noun_plural`)
requires scoping (`--words=…`) or clearing `wordForms` for the target rows.

### 2. Data layer — per-segment context signals

These live on the **example-sentence jsonb** (one object per sentence, inside the
entry's `exampleSentences` array). They are what tell the runtime *which* form to
pick. All three are **per-segment** (keyed by GSA segment string) and are produced
by the tagging pass (`tagSentenceSegments`) in
**`server/scripts/backfill/chinese/backfill-example-sentences.js`**.

| Field | Shape | Granularity | Drives |
|---|---|---|---|
| `partOfSpeechDict` | `Record<segment, posTag>` | Per-segment | Which `wordForms` key family to use |
| `numberDict` | `Record<segment, 'singular' \| 'plural'>` | **Per-noun-segment** | Noun singular vs. plural selection |
| `tenseDict` | `Record<segment, 'past' \| 'present' \| 'future'>` | **Per-verb-segment** | Verb tense-form selection |

**Why tense and number are both per-segment** — grammatical form is a property of
each individual word, not of the sentence. One sentence can contain a singular and
a plural noun (`I put the book on the shelves`), and equally two verbs in different
tenses (`I bought books, will return them tomorrow`). A sentence-level field for
either would be wrong for the second word, so `numberDict` and `tenseDict` both
mirror `partOfSpeechDict`'s per-segment shape and are written by the same tagging
pass. (`tense` was historically a single sentence-level generation field; it was
moved to the per-verb `tenseDict` — see the migration note below.) Each tagging
branch is gated on the segment's own POS: `numberDict` only for `noun`, `tenseDict`
only for `verb`/`auxiliary verb`.

These fields are stored verbatim on the sentence jsonb. At read time the
enrichment (`DictionaryDAL.enrichExampleSentencesMetadataBatch`) spreads
`{ ...sentence, _segments, segmentMetadata }`, so they all flow to the client
untouched — no extra plumbing per field.

### 3. Presentation layer — `resolveWordForm` (runtime selection)

`src/components/SegmentedSentenceDisplay.tsx`. When building each segment's popup
definition, if the tapped segment's dictionary metadata carries `wordForms` and the
sentence has a `partOfSpeechDict`, `resolveWordForm` chooses the form:

```ts
resolveWordForm(wordForms, pos, tense, number):
  verb / auxiliary verb + tense  → wordForms[tense]      ?? wordForms[pos]
  noun + number === 'plural'     → wordForms['noun_plural'] ?? wordForms['noun'] ?? wordForms['gerund']
  noun (singular/unspecified)    → wordForms['noun']        ?? wordForms['gerund']
  otherwise                      → wordForms[pos]
```

- `pos`    = `sentence.partOfSpeechDict[segment]`
- `tense`  = `sentence.tenseDict?.[segment]` (per-verb-segment)
- `number` = `sentence.numberDict?.[segment]` (per-noun-segment)

The `noun → gerund` fallback handles a verb used nominally in this sentence (tagged
`noun`), which has no dedicated `noun` key in a verb entry's `wordForms`.

Selection is skipped when the segment is annotated as a particle/classifier — that
contextual definition wins instead.

## Data flow (end to end)

```
backfill-word-forms.js ──► dictionaryentries_zh.wordForms        (per-headword form inventory)
backfill-example-sentences.js ──► exampleSentences[].{partOfSpeechDict, numberDict, tenseDict, senseDict}  (tagging pass)

   read path: DictionaryDAL.enrichExampleSentencesMetadataBatch
     • segmentWithDict()        → _segments
     • buildSegmentMetadata({ includeWordForms: true })
                                → segmentMetadata[seg].wordForms   (from the segment's own det row)
     • { ...sentence }          → partOfSpeechDict / numberDict / tenseDict pass through

   client: SegmentedSentenceDisplay
     • per segment: resolveWordForm(meta.wordForms, pos, tense, number) → popup definition
```

Key files:

| File | Role |
|---|---|
| `server/scripts/backfill/chinese/backfill-word-forms.js` | Generates `wordForms` (incl. `noun_plural`) |
| `server/scripts/backfill/chinese/backfill-example-sentences.js` | Tagging pass generates `partOfSpeechDict` / `numberDict` / `tenseDict` / `senseDict` |
| `server/dal/implementations/DictionaryDAL.ts` | `enrichExampleSentencesMetadataBatch` — attaches `wordForms` to segment metadata, passes context fields through |
| `server/dal/shared/segmentString.ts` | `buildSegmentMetadata` — copies each segment's `wordForms` onto its rendered metadata |
| `src/components/SegmentedSentenceDisplay.tsx` | `resolveWordForm` — runtime form selection in the popup |
| `server/types/index.ts`, `src/types.ts` | `exampleSentences` shape (incl. `numberDict`) |

## Backfill ordering

When adding/changing a form (e.g. a new `wordForms` key or a new context signal),
run **word-forms first**, then example-sentences, so the form a sentence's context
points at already exists:

1. `backfill-word-forms.js` — populate the form inventory. Re-runs need scoping
   because of the `WHERE "wordForms" IS NULL` guard.
2. `backfill-cluster-definitions.js` — must also precede example-sentences, which
   reads `definitionClusters` for each sentence's `sense` and skips unclustered rows.
3. `backfill-example-sentences.js [--all-discoverable]` — attach/refresh the
   per-sentence context fields.

Both scripts carry a `SCRIPT_VERSION`; bump it when the prompt/logic changes so the
run-log reflects which version produced a row.

## Extension points

- **New inflected form** — add the key to `ALLOWED_KEYS` + the prompt in
  `backfill-word-forms.js`, add a branch in `resolveWordForm`, and (if it needs a
  new context signal) add a per-token dict to the example-sentence script + the
  `exampleSentences` types, following the `numberDict` precedent.
- **Spanish form modification** — out of scope today. Requires an `es` word-forms
  backfill and including `wordForms` in `enrichSpanishExampleSentencesMetadataBatch`.
