# Backfill Enrichment Script - Setup & Troubleshooting

## Overview
The `backfill-enrichment.js` script populates enrichment data (synonyms, example sentences, parts of speech, and expansions) for existing Chinese vocabulary entries using Claude AI.

**Script Location:** `server/scripts/backfill-enrichment.js`

## Running the Script

### From Backend Container
```bash
docker exec cow-backend-local sh -c 'npx tsx /app/scripts/backfill-enrichment.js'
```

**Important Notes:**
- Use `sh -c` to avoid Git Bash path translation issues on Windows
- Use `npx tsx` instead of `node` because the import statements use TypeScript modules (`.ts` imports)
- The `--` flag is not needed

## Expected Output

The script processes vocabulary entries in phases:

1. **Enrichment Phase:** Processes all Chinese entries, adding:
   - Synonyms (from dictionary lookups)
   - Example sentences (generated by Claude)
   - Parts of speech (extracted by Claude)
   - Expansions (character expansions by Claude)

2. **expansionMetadata Phase:** Updates metadata for expansions (typically 0 entries if already populated)

3. **Tone Phase:** Backfills missing tone data (typically 0 entries if already populated)

**Example Output:**
```

📊 Found 130 Chinese entries to process
[1/130] Processing: 今天...
  ✓ Updated with 0 synonyms, 3 examples, 0 POS, expansion: null
...
============================================================

Succeeded: 130 (100%)

```

## Expected Results

When all entries are processed:
- **Total entries:** 135
- **Entries with expansions:** ~5 (3-5%)
- **Entries with NULL expansions:** ~130 (95-97%)

**This is normal behavior.** Most Chinese words cannot be meaningfully expanded by adding characters between their original characters. Claude returns `null` for these words, which is the correct behavior.

### Verification Query
```sql
SELECT
  COUNT(*) as total_entries,
  COUNT(expansion) as entries_with_expansion,
  COUNT(CASE WHEN expansion IS NULL THEN 1 END) as entries_with_null_expansion
FROM vocabentries;
```

## Common Issues & Solutions

### Issue 1: Missing Database Columns

**Error:**
```
Database error: error: column "discoverable" does not exist
```

**Cause:** The `dictionaryentries` table schema is missing enrichment-related columns.

**Solution:** Add the missing columns to the database:

```bash
docker exec -e PGPASSWORD=cow_password_local cow-postgres-local psql -h localhost -U cow_user -d cow_db << 'EOF'
ALTER TABLE dictionaryentries ADD COLUMN discoverable BOOLEAN NOT NULL DEFAULT FALSE;
ALTER TABLE dictionaryentries ADD COLUMN script VARCHAR(20);
ALTER TABLE dictionaryentries ADD COLUMN "hskLevel" VARCHAR(10);
ALTER TABLE dictionaryentries ADD COLUMN breakdown JSONB;
ALTER TABLE dictionaryentries ADD COLUMN synonyms JSONB;
ALTER TABLE dictionaryentries ADD COLUMN "exampleSentences" JSONB;
ALTER TABLE dictionaryentries ADD COLUMN "partsOfSpeech" JSONB;
ALTER TABLE dictionaryentries ADD COLUMN expansion TEXT;
ALTER TABLE dictionaryentries ADD COLUMN "expansionMetadata" JSONB;
EOF
```

**Verification:**
```bash
docker exec -e PGPASSWORD=cow_password_local cow-postgres-local psql -h localhost -U cow_user -d cow_db -c "SELECT column_name FROM information_schema.columns WHERE table_name='dictionaryentries' ORDER BY ordinal_position;"
```

### Issue 2: TypeScript Import Path Errors

**Error:**
```
Error [ERR_MODULE_NOT_FOUND]: Cannot find module '/app/db.js'
```

**Cause:** Running the script with `node` instead of `tsx`. The script uses TypeScript imports (`import ... from '../db.js'`) which require the tsx runtime.

**Solution:** Use `npx tsx` to run the script:
```bash
docker exec cow-backend-local sh -c 'npx tsx /app/scripts/backfill-enrichment.js'
```

### Issue 3: Git Bash Path Translation

**Error:**
```
node:internal/modules/cjs/loader:1210
Error: Cannot find module '/app/C:/Program Files/Git/app/scripts/...'
```

**Cause:** Git Bash is translating `/app` paths to Windows paths.

**Solution:** Wrap the command in `sh -c` to avoid path translation:
```bash
docker exec cow-backend-local sh -c 'npx tsx /app/scripts/backfill-enrichment.js'
```

### Issue 4: Database Connection Issues

**Error:**
```
psql: error: connection to server failed: FATAL: role "postgres" does not exist
```

**Cause:** Using wrong PostgreSQL credentials.

**Solution:** Use the correct credentials for the local setup:
- Container: `cow-postgres-local`
- User: `cow_user`
- Password: `cow_password_local` (set via PGPASSWORD environment variable)
- Database: `cow_db`

```bash
docker exec -e PGPASSWORD=cow_password_local cow-postgres-local psql -h localhost -U cow_user -d cow_db -c "your_query"
```

## Notes

1. **Expansion Nulls Are Expected:** When Claude cannot meaningfully expand a word by adding characters between the originals, it correctly returns `null`. This is not a bug—it's the intended behavior.

2. **Idempotent:** The script can be run multiple times safely. It will update existing enrichment data if ANTHROPIC_API_KEY is set.

3. **API Key Required:** To regenerate expansions, set the `ANTHROPIC_API_KEY` environment variable in the backend container's `.env.docker`.
